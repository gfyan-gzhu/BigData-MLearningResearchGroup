# 基于错误标签纠正与动态调度的知识蒸馏工具

本项目实现了**错误标签纠正与动态权重调度的知识蒸馏（Knowledge Distillation）** 算法，针对文本情感分类任务优化了教师-学生模型的知识迁移过程。通过检测并纠正教师模型的错误预测标签，结合余弦退火的动态蒸馏权重策略，显著提升了轻量化BERT学生模型的分类性能，在保持模型推理速度的同时缩小了与教师模型的性能差距。

## 📋 项目概述
### 核心流程
1. **模型构建**：通过`mumodel.py`构建12层BERT教师模型和6/4层轻量化BERT学生模型，加载预训练权重初始化教师模型。
2. **数据预处理**：通过`dataload.py`完成数据集加载、分层划分、文本Token化和批量数据加载器构建。
3. **改进型蒸馏训练**：通过`kdtrain.py`执行蒸馏主流程，结合`tomethod.py`的核心算法，实现错误标签纠正与动态权重调度的蒸馏训练。
4. **模型评估与日志**：训练过程中自动验证并保存最优模型，通过`printlog.py`记录完整训练日志和实验结果。

### 核心特性
- 教师模型错误标签自动检测与概率分布校准，消除错误知识迁移
- 基于余弦退火的动态蒸馏权重调度，训练前期侧重易样本、后期侧重难样本
- 集成DKD解耦知识蒸馏损失，分离目标类与非目标类的知识迁移
- 完整的实验日志管理，自动记录训练配置、每轮指标和最终结果
- 支持梯度裁剪和权重衰减，防止模型过拟合

## 🚀 使用步骤

### 步骤1：数据集准备
准备情感分类数据集，保存为`data.csv`并放置在项目根目录，数据集必须包含以下两列：
- `title`：文本内容（字符串类型）
- `label`：情感标签（整数类型，从0开始编号）

> 提示：可在`dataload.py`中修改`data_dir`和`total_csv_name`参数指定自定义数据集路径，调整`batch_size`和`max_seq_len`适配硬件资源。

### 步骤2：预训练模型准备
下载BERT中文预训练模型文件，放置在`./model`目录下，目录需包含以下核心文件：
- `vocab.txt`：词表文件
- `pytorch_model.bin`：模型权重文件
- `config.json`：模型配置文件

> 提示：可使用HuggingFace Hub下载`bert-base-chinese`模型，或在`dataload.py`中修改`pretrained_model`参数指定其他预训练模型路径。

### 步骤3：配置训练参数
修改`kdtrain.py`中的核心训练参数以适配你的任务：
```python
# 任务核心参数
NUM_CLASSES = 10  # 情感分类类别数（根据你的数据集修改）
GLOBAL_MAX_SEQ_LEN = 128  # 文本最大序列长度
epochs = 100  # 总训练轮数

# 蒸馏超参数
T = 2.0  # 蒸馏温度系数
lambda_stu = 0.5  # 学生模型交叉熵损失权重
```
其他可调整的关键参数：
- `lr`：AdamW优化器学习率（默认：2e-5）
- `weight_decay`：权重衰减系数（默认：1e-4）
- `warmup`：DKD损失预热轮数（默认：20）

### 步骤4：启动蒸馏训练
执行训练主程序：
```bash
python kdtrain.py
```
#### 训练过程说明：
1. 自动加载数据集并完成6:2:2分层划分，输出各数据集样本数量
2. 加载预训练权重初始化教师模型并冻结，初始化学生模型
3. 每轮训练完成后在验证集上评估准确率，自动保存最优模型权重
4. 训练结束后自动加载最优模型，在测试集上进行最终评估并输出结果

## ⚠️ 注意事项
1. **类别数匹配**：必须将`kdtrain.py`中的`NUM_CLASSES`修改为你的数据集实际类别数，否则会出现维度不匹配错误。
2. **GPU设备指定**：如需指定特定GPU运行，可在`kdtrain.py`开头添加`os.environ["CUDA_VISIBLE_DEVICES"] = "0"`（数字为GPU编号）。
3. **学生模型选择**：如需使用4层更轻量化的学生模型，将`kdtrain.py`中的`model_small_6`替换为`model_small_4`，并修改模型保存路径。
4. **损失权重调优**：可根据任务特点调整`lambda_stu`参数，当学生模型与教师模型差距较大时，可适当增大该值（如0.6-0.7）。
5. **梯度裁剪**：代码已启用梯度裁剪（最大范数1.0），可有效防止训练过程中梯度爆炸，无需额外修改。

## 📈 核心算法逻辑
### 错误标签校准模块（tomethod.py）
- `check()`：对比教师模型预测标签与真实标签，生成错误样本掩码
- `calibrate_target_weight()`：对教师预测错误的样本，结合真实标签校准教师输出概率分布，通过动态系数`alpha`平衡教师预测与真实标签的权重
- 权重约束在[0,1]范围内，保证校准后的概率分布合法有效

### 动态蒸馏调度模块（tomethod.py）
- `lr_cosine()`：基于余弦函数生成动态权重系数`gamma`，随训练轮次从0线性增长至1
- 总蒸馏损失 = (1-gamma)×易样本蒸馏损失 + gamma×难样本蒸馏损失
- 实现训练过程中从易到难的知识迁移，提升难样本的学习效果

### DKD解耦损失模块（tomethod.py）
- 实现`DKDLoss`类，将传统KD损失解耦为目标类蒸馏损失（TCKD）和非目标类蒸馏损失（NCKD）
- 通过`alpha`和`beta`参数分别控制两部分损失的权重
- 支持预热机制，训练前期逐步增加DKD损失的权重，提升训练稳定性
